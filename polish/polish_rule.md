# 《计算机入门学习》内容打磨规则

> 面向零基础读者，目标是让"第一次碰电脑的人"也能独立读懂、跟着操作。打磨时把自己当成完全不懂电脑的新手——不知道右键是什么、没听过任务栏、看到快捷键就发慌。

---

## P0：底线要求（发布前必须全部通过）

### 安全与准确性
- [ ] **技术操作准确**：步骤在最新 Windows 版本验证过，命令行代码正确（如 `ipconfig` 不是 `ipconfig.exe`），单位换算正确（1 GB = 1024 MB，厂商按 1000）
- [x] **命令行安全警告**：涉及命令行的章节，开头必须有总体安全提示；`del`、`format`、`reg delete` 等危险命令旁必须有单独警告
- [x] **不恐吓读者**：不说"会损坏电脑"，说"有风险"；不说"必须"，说"建议"
- [x] **看不懂就停手**：操作指导中必须提醒"看不懂就停手，问懂的人"或"先搜索确认再操作"

### 新手不迷路
- [x] **阅读前提**：每章开头用 `> 阅读前提：` 说明需要的前置知识，引用具体章节名（如"参见《认识PC》"）
- [x] **不假设已知**：不引用未解释的术语；命令行操作前给出"按 Win+R 输入 cmd"的完整步骤
- [x] **概念闭环**：每个新概念有"是什么→为什么→怎么用"，配合类比（如"内存=工作桌"）和具体例子
- [x] **兜底方案**：复杂操作给出"先重启试试"/"需要额外操作"/"找人帮忙"的分流思路

## P1：核心体验（严重影响阅读流畅度）

### 语气与口吻
- [ ] 用"你"直接对话，避免"用户"等第三人称
- [ ] 耐心解释，不假设读者已知任何概念
- [ ] 避免命令式，改用建议式（"你可以..."而非"你要..."）
- [ ] 鼓励为主，适当加入成就感提示（尤其是安全章节后）
- [ ] **避免居高临下的措辞**：不说"很多人不知道""很多新手以为"，改为"你可能还没注意到"或直接陈述事实
- [ ] **避免规范性压力**：不说"应该掌握""必须学会"，改为"养成习惯后能帮你..."

### 格式规范
- [ ] 标题层级：`#` 书名 → `##` 章节 → `###` 小节 → `####` 细分
- [ ] 步骤用编号列表（1. 2. 3.），非步骤用项目符号（-）
- [ ] 引用块分层：
  - `> 阅读前提：` — 章节前置条件
  - `> **XXX**：` — 重点提示或警告（加粗）
  - `> ` 普通引用 — 补充说明或总结
- [ ] 无 emoji、无花哨视觉标记；允许基础排版 HTML（如 div 居中），禁止装饰性符号
- [ ] 代码/按键用反引号（`Ctrl+C`），路径用反引号（`C:\Users`）
- [ ] 代码块标注正确语言类型（`cmd`、`text`、`path` 等）
- [ ] **引号统一**：全文使用中文弯引号 `""`，不使用直角引号 `「」`

### 术语与缩写
- [ ] 首次出现专业术语时简要解释
- [ ] 中英文对照：中文名（英文名），如"个人计算机（PC）"
- [ ] 避免缩写（不用"UI"，用"界面"；不用"OS"，用"操作系统"）
- [ ] **术语一致性**：同一概念全书使用统一名称（如"文件资源管理器"而非"文件管理器"）

### 系统版本适配
- [ ] 优先以 Windows 11 为默认描述
- [ ] Windows 10 差异用 `[查]` 标记（紧跟句子末尾，前面不加空格），提示搜索最新教程
- [ ] `[查]` 也用于任何因版本/界面差异可能不同的操作

### 快捷键规范
- [x] 《基本操作》首次介绍全局快捷键，后续章节首次使用某快捷键时必须提供菜单替代方式
- [ ] "快速路线"章节（1-3章）不得依赖"深度管理"阶段的快捷键

## P2：质量提升（影响专业感和 completeness）

### 内容完整性
- [ ] 操作步骤编号清晰，有"快速排查"和"进阶排查"分层
- [ ] 有"试一试"或"现在动手"的引导
- [ ] 预判常见错误（如"如果找不到设置"）
- [ ] 单章全新概念控制在 5-8 个以内，超出建议拆分
- [ ] 每章至少包含一个"现在就能试"的操作或观察练习
- [ ] **不假设前置知识跨章存在**：引用前面章节的概念时，确认该概念确实在前置章节中出现过

### 结构与衔接
- [ ] "快速路线"章节的阅读前提只依赖快速路线内的章节
- [ ] 章节顺序符合学习曲线
- [ ] 避免重复（如快捷键只在《基本操作》讲一次）
- [ ] 适当留白，不过度堆砌信息
- [ ] 有总结/回顾帮助记忆（表格或要点列表）
- [ ] **章节编号连续**：检查 `##` 级标题编号无重复、无遗漏

### 章节长度
- [ ] 快速路线章节：3000-6000 字
- [ ] 常规章节：3000-8000 字
- [ ] 深度/附录章节：可放宽至 10000-12000 字，过长建议拆分

### 可检索性
- [ ] 标题包含关键词（方便搜索定位）
- [ ] 术语表格式统一：术语 | 一句话解释 | 相关章节

---

## P3： polish 细节（锦上添花）

### 跨章节链接
- [ ] 引用其他章节时，链接文本统一用书名号《》包裹，如 `[《电脑配置详解》](../扩展知识/电脑配置详解.md)`
- [ ] **链接路径正确**：验证相对路径指向的文件确实存在
- [ ] **链接文本与实际标题一致**：如链接文本"理解你的网络"而非"网络进阶"

### 延伸阅读
- [ ] 需要时添加"延伸阅读"区块，标注来源和适用场景
- [ ] 允许引用外部图文资源作为补充，正文本身不使用截图，优先用文字描述或表格替代
- [ ] **不使用简称**：如"计课"应写全称《你缺失的那门计算机课》

### 特殊内容格式
- [ ] 软件推荐类包含：解决什么问题 / 为什么推荐它 / 官网地址 / 不足之处
- [ ] 官网地址格式：`[官网](https://...)` 内嵌链接，点击前可核对域名

### 通读自检
- [ ] 按"快速路线"通读一遍，记录卡壳点
- [ ] 避免连续多章都是警告/注意事项，适当加入成就感提示
- [ ] 长段落拆分为短句，减少认知负担

## 附：新增检查项（基于两轮打磨实践）

### 技术准确性专项
- [ ] **硬件信息时效性**：CPU/显卡/内存等硬件型号和价格描述需定期更新
- [ ] **Windows 界面变更**：任务管理器选项卡名称、设置路径等可能随版本更新
- [ ] **命令行工具默认行为**：Win+X 菜单、Print Screen 键等默认行为可能变化

### 写作一致性专项
- [ ] **Wi-Fi 写法统一**：使用 "Wi-Fi"（带连字符），不使用 "WiFi"
- [ ] **术语全书统一**：建立术语表，确保同一概念在不同文件中使用相同名称
- [ ] **阅读前提句式统一**：建议统一为"你已经了解了..."

### 新手友好性专项
- [ ] **高频操作入口必须教**：如"设置"应用（Win+I）、搜索框（Win+S）等全书频繁引用的操作，需在首次出现前教会读者
- [ ] **避免信息过载**：密集概念之间加过渡句；命令列表按场景分组
- [ ] **抽象概念配例子**：权限规则、编码原理等抽象内容给出具体场景示例
