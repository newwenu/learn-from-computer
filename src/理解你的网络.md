# 理解你的网络：从插上网线到打开网页发生了什么

> 阅读前提：你已经了解了操作系统的基本概念（参见《理解你的操作系统》）。本文讲的是网络怎么工作，不涉及具体操作步骤（Wi-Fi 连接等已在《日常使用》中讲过）。
>
> 本文讲解概念，不需要跟着操作。如果某些部分读起来吃力，可以先跳过，等有了更多使用经验后再回来看。

---

## 一、网络是什么

把两台电脑用一根网线连起来，再配好网络设置，它们就能互相传数据——这是网络最简单的一种形式。

**互联网**就是把全世界无数台这样的电脑和网络设备连接在一起。你家的电脑 → ... → 目标网站的服务器，中间经过十到二十个节点，全过程一般在几十到几百毫秒内完成。

关键要理解一件事：**最常见的网络场景是一台电脑（你的设备）和另一台电脑（服务器）之间传输数据**。刷网页、看视频、打游戏——这些常见应用的本质都是"你的设备"和"某台服务器"之间在收发数据。

## 二、IP 地址：网络上的门牌号

每台上网的设备都需要一个唯一的地址，其他设备才能找到它。这个地址就是 **IP 地址**。

### IPv4：最常见的 IP 地址

格式是四组数字，用点隔开，就像 `192.168.1.10`。

每一组取值范围 0-255。IPv4 总共约 43 亿个地址，数量看起来很多，但全球设备远超这个数。不过不用担心——现在的网络技术（比如家里的路由器让多台设备共享一个 IP 上网）已经缓解了地址不够的问题，日常上网基本不受影响。IPv6 是下一代地址格式，数量更多，正在逐步普及。

### IPv6：下一代地址

格式比 IPv4 更长，数量多到给地球上每粒沙子都能分一个。国内主流运营商已在大部分地区支持，多数家用路由器也默认开启了 IPv6（前提是运营商提供）。**日常使用不需要手动选择——系统会自动判断用哪种**。

### 公网 IP vs 内网 IP

这是一个特别容易混淆的概念。

**公网 IP**：全球唯一的地址，可以直接从互联网访问到你的设备。家庭宽带的公网 IP 通常是动态的（运营商不定期更换）。IPv4 资源紧缺，所以普通家庭宽带大多不分配公网 IPv4。

**内网 IP**：只在你的局域网内有效的地址。典型的内网 IP 段有 `192.168.x.x`、`10.x.x.x`、`172.16.x.x ~ 172.31.x.x`。你在手机或电脑上看到的"IP 地址"，绝大多数情况下是内网 IP——是家里的路由器分配给你的，互联网上的其他设备无法直接用这个地址找到你。

**判定方法**：打开命令提示符（`Win` + `R` → 输入 `cmd` → 回车），输入 `ipconfig`，找到"IPv4 地址"那行。如果以 `192.168.`、`10.` 或 `172.16.` ~ `172.31.` 开头，就是内网 IP。想知道你的公网 IP，可以在浏览器里搜索"IP"——搜索引擎会直接显示你当前的公网 IP。

### 端口：门牌号之后的房间号

一台服务器可以用一个 IP 地址同时提供多种服务（网页、邮件、文件传输），靠的就是端口号来区分。

- IP 地址：`192.168.1.10`（门牌号，定位到这台设备）
- 端口：`:80`（房间号，定位到这个服务）

常见端口：

| 端口 | 用途 |
|------|------|
| 80 | HTTP（普通网页） |
| 443 | HTTPS（加密网页） |
| 25 | SMTP（发邮件） |

你在浏览器地址栏看到的 `https://www.baidu.com` 实际上相当于 `https://www.baidu.com:443`，只是浏览器默认帮你省略了 443 端口。平时你看不到端口号，因为浏览器会自动补上常用的默认端口。

什么时候你会看到端口号？比如你自己搭建了一个网站测试服务器，地址可能是 `http://192.168.1.10:8080`——这里的 `:8080` 就是手动指定的端口，告诉浏览器"去这台设备的 8080 号房间找服务"。

## 三、DNS：把名字翻译成地址

人类记名字（`baidu.com`），电脑记 IP 地址（`39.156.66.10`）。DNS（域名系统）就是负责把域名翻译成 IP 地址的"电话本"。

### 当你输入 baidu.com 按回车，背后发生了什么

1. 浏览器先检查本地缓存——之前查过的域名会暂时记住
2. 如果本地没有，向操作系统配置的 DNS 服务器查询（通常是你家路由器转发给运营商 DNS 服务器，也可能是你手动设置的 114.114.114.114 或 8.8.8.8）
3. 这个 DNS 服务器会先查自己的缓存，如果也没有，它就扮演"代办员"的角色，替你逐级去问：
   - 先问**根域名服务器**（DNS 查询链的起点，知道所有顶级域名服务器在哪）
   - 根服务器告诉它 `.com` 顶级域名服务器在哪
   - 再问 `.com` 顶级域名服务器，`baidu.com` 的 DNS 服务器在哪
   - 最后问 `baidu.com` 的 DNS 服务器，`www.baidu.com` 的 IP 地址是多少
4. 拿到 IP 地址 `39.156.66.10` 后，DNS 服务器把结果返回给你的电脑，浏览器开始建立连接

整个过程通常在几十到一百多毫秒内完成。DNS 服务器会把查到的结果缓存一段时间，下次有人问同一个域名时直接回答，不用再去问一遍。如果 DNS 服务器响应慢或返回错误 IP，就会出现"能连上微信但打不开网页"这类现象——你电脑的网络是通的，但域名解析出了问题。

### 常见 DNS 问题

- 某个网站打不开但其他正常 → 可能是这个域名的 DNS 解析临时出错
- 所有网站都打不开但微信能上 → 可能是 DNS 配置有问题（微信客户端有自己的域名解析机制，不依赖系统配置的 DNS 服务器，因此当系统 DNS 故障时微信仍能正常工作）
- 某个网站被劫持到广告页面 → 可能是路由器或 DNS 被篡改

**解决办法**：把 DNS 改成公共 DNS，比如 `114.114.114.114`（国内，速度快）。在国内网络环境下，`8.8.8.8` 和 `1.1.1.1` 的稳定性和速度通常不如国内 DNS，建议优先使用国内公共 DNS。具体操作方法可以搜索"Windows 修改 DNS 地址"。

### 补充：网站打不开的几个常见原因

除了 DNS 问题，一个网站打不开还可能有以下几种情况：

| 原因 | 怎么判断 |
|------|---------|
| 网站服务器挂了 | 用手机流量试试——手机能打开，说明是你网络环境的问题；手机也打不开，就是网站本身的问题 |
| 浏览器或安全软件拦截 | 极少见。换一个浏览器试一下就知道是不是浏览器本身的问题了 |
| 你的网络环境对该网站有限制 | 遇到这种情况，一般人没有绕过的方法。在搜索引擎里搜一下网站名 +"打不开"，看看是不是很多人都遇到了 |

**最简单的判断方法就是换一个网络试一下**：手机切到流量 vs 连着你家 Wi-Fi，分别打开同一个网站，结果一对比就知道问题出在你自己这边还是网站那边。

## 四、路由器与家庭网络

### 路由器是干什么的

绝大多数家庭网络结构是这样的（光猫和路由器可能是两台独立的设备，也可能是合二为一的一体机）：

```text
互联网 ── 光猫 ── 路由器 ── 你的设备（手机/电脑/电视）
```

路由器主要有三个功能：

**1. NAT（网络地址转换）**：你的设备只有内网 IP（如 `192.168.1.10`），路由器把内网 IP 转换成自己的公网 IP 去访问互联网，再把回来的数据转发给正确的设备。一台路由器能让全家十几台设备共享一个公网 IP 上网。NAT 就像公司前台的代收服务：包裹（网络数据）统一送到前台（路由器的公网 IP），前台再根据收件人（内网 IP）分发给对应工位。

**2. DHCP（动态主机配置协议）**：设备连上路由器时，路由器自动给它分配一个内网 IP、子网掩码、默认网关和 DNS 服务器。所以你插上网线或连上 Wi-Fi 就能直接用，不需要手动设 IP——**这就是为什么家用网络通常不需要自己配 IP 地址**。

**3. 无线接入点**：发射 Wi-Fi 信号。

### Wi-Fi 2.4GHz 和 5GHz

如今大多数路由器支持双频 Wi-Fi，两个频段的区别：

| 频段 | 穿墙能力 | 速度 | 干扰 |
| ------ | --------- | ------ | ------ |
| 2.4GHz | 强 | 较慢（普通设备通常 72-300Mbps，新设备在此频段可略高） | 严重（蓝牙、微波炉、邻居 Wi-Fi 都用这个频段） |
| 5GHz | 弱 | 快（理论上最高数 Gbps） | 少 |

**日常建议**：在路由器旁边用 5GHz，隔了墙信号不好时切到 2.4GHz。很多路由器支持"双频合一"（一个 SSID——Wi-Fi 网络名称——自动切换），但有时切换不灵敏，分开命名更可控。

### 插网线 vs Wi-Fi

网线的优势是稳定、延迟低、干扰少。如果条件允许（路由器在电脑旁边），打游戏或视频会议时插网线体验更好。Wi-Fi 的优势是方便、无束缚。

### 子网掩码与网关（知道有这回事就行）

除了 IP 地址，每台设备还有两个参数：

- **子网掩码**（通常是 `255.255.255.0`）：配合 IP 地址使用，帮你判断两台电脑是不是在同一个局域网里。日常使用不需要手动设置——DHCP 自动分配好了。

- **默认网关**（通常是 `192.168.1.1` 或 `192.168.0.1`）：局域网通往互联网的出口，就是你的路由器 IP。想跟外网通信时，数据先发给网关，由它转发出去。

这两个参数在 `ipconfig` 里都能看到，日常使用不需要手动设置——DHCP 自动分配好了。如果你在折腾网络时不小心改乱了，在网络设置里选"自动获取 IP 地址"和"自动获取 DNS 服务器地址"就能恢复。

## 五、带宽、网速与延迟

### Mbps 和 MB/s 的区别

这是最容易被误解的概念。

- 宽带的单位是 **Mbps**（兆比特每秒，Megabits per second）
- 下载速度的单位是 **MB/s**（兆字节每秒，Megabytes per second）

**1 字节 = 8 比特**（1 B = 8 b），所以：

- 你家的 100Mbps 宽带，理论最高下载速度 = 100 ÷ 8 = **12.5 MB/s**
- 200Mbps 宽带 → 25 MB/s
- 500Mbps 宽带 → 62.5 MB/s
- 1000Mbps（千兆）→ 125 MB/s

所以当运营商说"100 兆宽带"，你下载文件最高看到 10-12 MB/s 是正常的，不是网速有问题。实际速度还会因为网络损耗、服务器限速等因素再低一些，尤其在 Wi-Fi 下损耗更大（能到理论值的 50-70% 对 Wi-Fi 来说已属正常，有线连接通常 80% 以上）。

### 带宽 vs 延迟

两个容易混淆的概念：

| 概念 | 比喻 | 影响什么 |
| ------ | ------ | --------- |
| **带宽** | 水管粗细 | 同时能传多少数据——影响下载速度、视频清晰度 |
| **延迟** | 水流从源头到终点的时间 | 响应快慢——影响游戏手感、网页打开速度、视频通话 |

带宽越大，下载越快（前提是对方服务器不限制速度）；延迟越低，操作越跟手。**两者不直接相关**——你可能带宽很大（500Mbps）但延迟很高（打游戏卡顿）；也可能带宽很小（20Mbps）但延迟极低（游戏流畅）。

延迟的单位是**毫秒（ms）**：

- 局域网内：< 1ms
- 同城网络：3-15ms
- 国内跨省：10-50ms
- 跨国：100-300ms
- 超过 100ms 游戏中就能感到延迟，超过 200ms 视频通话会有明显顿挫感

游戏玩家常说的 ping 值就是延迟——ping 50ms 表示数据包往返一次需要 50 毫秒。

### 实际网速受什么影响

你家宽带 500Mbps，不代表你每次下载都能跑到 500Mbps。实际速度取决于以下环节中的最慢一环：

1. **宽带套餐上限**（运营商给你设的速度上限）
2. **路由器性能**（百兆路由器跑不满 200M 宽带）
3. **网线质量**（劣质网线只能跑百兆）
4. **Wi-Fi 信号**（隔墙后速度大幅下降）
5. **目标服务器限速**（对方只给你 10MB/s 的上传速度，你家再大带宽也没用）
6. **同时使用的设备数**（全家都在看视频，带宽被分了）

## 六、常见网络问题排查

> 这节用到命令行操作，如果你还不熟悉命令提示符，可以先阅读《命令行入门》。本节只涉及几个简单命令，跟着敲就行。

### 1. ping —— 检查目标能不能通

按 `Win` + `R` → 输入 `cmd` → 回车，在黑窗口里输入：

```cmd
ping baidu.com
```

如果返回类似于 `来自 39.156.66.10 的回复: 字节=32 时间=12ms TTL=52`，说明网络基本连通。`时间=`后面的数字就是延迟。

如果返回 `请求超时` 或 `找不到主机`，说明网络可能有问题——也可能是对方服务器没有配置回应 ping 请求，不代表完全连不上。

**实用场景**：

- 打游戏卡顿 → `ping baidu.com -t`（加 `-t` 会持续 ping，看延迟是否忽高忽低；按 Ctrl+C 停止），能反映到你 ping 的目标服务器的网络稳定性
- 怀疑 DNS 有问题 → `ping 114.114.114.114`（直接 ping IP，跳过了 DNS 解析），能通说明到该 IP 的网络连接正常，问题可能在 DNS
- 连着 Wi-Fi 但上不了网 → `ping baidu.com`，不通就说明网络确实断了

### 2. ipconfig —— 查看本机网络配置

```cmd
ipconfig
```

会列出你电脑上所有网络适配器的信息：

| 信息项 | 看什么 |
| ------ | ------ |
| IPv4 地址 | 你的内网 IP |
| 子网掩码 | 通常是 255.255.255.0 |
| 默认网关 | 路由器的 IP |
| DNS 服务器 | 当前用的 DNS |

`ipconfig /flushdns` 可以清空 DNS 缓存，如果你改了 DNS 设置后没生效，可以跑一下这个命令。

### 3. tracert —— 看数据包经过哪些节点

```cmd
tracert baidu.com
```

会列出从你电脑到目标服务器经过的每一跳（路由器/网关），每行一个节点。当你感觉"网络慢"时，这个命令能帮你看出来是卡在哪一跳。如果中间某一行显示"请求超时"，不要慌——这只是那个节点不想理你，不代表你的网断了。如果某一跳长时间没响应或者延迟突然变得很高，问题可能就出在这里。

### 4. 断网排查三步走

遇到"连不上网"，按这个顺序来：

1. **看是全部打不开还是某个网站打不开**
   - 全部打不开 → 检查网络连接本身（Wi-Fi 是否连上、路由器是否亮灯）
   - 某个网站打不开 → 一般是 DNS 或网站本身的问题

2. **ping 网关**（`ping 192.168.1.1` 或你的路由器 IP）
   - 通 → 你到路由器的连接是好的，问题在路由器之外
   - 不通 → 问题在电脑到路由器之间（Wi-Fi 断开、网线松了、路由器死机）

3. **ping 公网 IP**（`ping 114.114.114.114`）
   - 通 → 网络是通的，问题在 DNS（`ping baidu.com` 不通但 `ping 114.114.114.114` 通，就是 DNS 问题）
   - 不通 → 路由器到互联网断了（重启光猫和路由器，还不行联系运营商）

### 注意：不建议安装"网络加速器""网速优化"类软件

Windows 系统本身的网络栈是经过充分优化的。所谓的"网络加速"软件要么没实际效果，要么夹带弹窗广告。如果网速慢，先排查物理原因（宽带套餐、路由器、网线、Wi-Fi 信号），不要寄希望于软件"优化"。只有游戏加速器是例外——它把你的游戏数据通过专用线路转发到游戏服务器，而不是走常规网络路径，来降低游戏延迟。这和上述"优化软件"的工作原理完全不同。

## 七、总结

| 概念 | 一句话理解 |
| ------ | --------- |
| IP 地址 | 设备在网络上的门牌号 |
| 端口 | 同一设备上不同服务的房间号 |
| DNS | 把域名翻译成 IP 地址的电话本 |
| 路由器 | 连接家庭网络和互联网的中转站 |
| NAT | 让多台设备共享一个公网 IP |
| DHCP | 设备连上网络时自动分配配置参数 |
| 带宽 | 水管粗细，决定下载速度上限 |
| 延迟 / ping | 数据往返耗时，决定响应快慢 |
| Mbps vs MB/s | 1 MB/s = 8 Mbps，宽带的"兆"和下载的"兆"差了 8 倍 |

---

自行探索：

1. **查看你电脑的完整网络路径**：`tracert baidu.com`，数一下经过了多少跳
2. **查看当前网络连接列表**：`netstat -n` 会列出你电脑上当前活跃的网络连接。你会看到很多 `xxx.xxx.xxx.xxx:443` 这样的条目——每一行都是一台服务器
3. **公共 DNS 地址**：114.114.114.114（国内，速度快）、223.5.5.5（阿里 DNS）、119.29.29.29（腾讯 DNS）。国外的 8.8.8.8 和 1.1.1.1 在国内经常被干扰，不推荐作为首选。
