# 网络

## 基本概念

- **IP 地址**：设备在网络中的唯一标识，如 `192.168.1.100`
- **DNS**：域名系统，将网址（如 `baidu.com`）转换为 IP 地址（如 `220.181.38.148`），DNS 异常时网址无法打开但 IP 可直接访问
- **网关**：连接内网和外网的设备地址，通常是路由器的 IP，如 `192.168.1.1`
- **Mbps**：兆比特每秒，带宽单位；1 字节（Byte）= 8 比特（bit），100Mbps 实际下载速度约 12.5MB/s
- **MAC 地址**：网卡出厂时烧录的物理地址，由 6 组十六进制数组成（如 `1A:2B:3C:4D:5E:6F`），全球唯一，用于局域网内设备识别
- **端口号**：区分同一设备上不同网络服务的编号，范围 0-65535；常见端口号：80（HTTP）、443（HTTPS）、21（FTP）、22（SSH）、3389（远程桌面）

## 网速换算

`实际下载速度（MB/s）= 宽带带宽（Mbps）÷ 8`

| 宽带带宽 | 理论最大下载速度 |
|----------|------------------|
| 100Mbps | 12.5MB/s |
| 200Mbps | 25MB/s |
| 500Mbps | 62.5MB/s |
| 1000Mbps | 125MB/s |

## IPv4 与 IPv6

| 对比项 | IPv4 | IPv6 |
|--------|------|------|
| 地址长度 | 32 位 | 128 位 |
| 地址格式 | `192.168.1.1` | `2400:3200::1` |
| 地址数量 | 约 43 亿 | 约 3.4×10³⁸ |
| 本机回环地址 | `127.0.0.1` | `::1` |

查看本机 IPv6 地址：`ipconfig /all`，找到"IPv6 地址"字段

## Wi-Fi 频段

| 频段 | 穿墙能力 | 速度 | 干扰 |
|------|----------|------|------|
| 2.4GHz | 强 | 最大 300Mbps | 多（蓝牙、微波炉等） |
| 5GHz | 弱 | Wi-Fi 5 最大 1.3Gbps，Wi-Fi 6/6E 可达数 Gbps | 少 |

## 断网排查步骤

1. **判断范围**：所有设备都断网 / 只有当前设备
   - 所有设备：检查路由器、光猫、网线
   - 只有当前设备：检查 Wi-Fi 开关、网卡驱动、IP 地址
2. **重启设备**：重启路由器、光猫、当前设备
3. **检查物理连接**：网线是否插紧、Wi-Fi 信号是否正常
4. **测试连通性**（命令行）：
   - `ipconfig` 查看默认网关
   - `ping 默认网关`（如 `ping 192.168.1.1`），测试内网连通性
   - `ping 223.5.5.5`（公共 DNS），测试外网连通性
   - `ping baidu.com`，测试 DNS 解析
5. **修改 DNS**：如果 DNS 解析失败，尝试修改公共 DNS

## DNS 配置

### 修改步骤

1. 打开"设置→网络和 Internet→高级网络设置→更多网络适配器选项"
2. 右键当前网络连接>"属性"
3. 双击"Internet 协议版本 4 (TCP/IPv4)"
4. 选择"使用下面的 DNS 服务器地址"
5. 输入首选和备用 DNS 地址
6. 点击"确定"

## 常用网络命令

| 命令 | 作用 | 常用示例 |
|------|------|----------|
| `ipconfig` | 查看网络配置 | `ipconfig`、`ipconfig /all`、`ipconfig /flushdns` |
| `ping` | 测试连通性 | `ping baidu.com`、`ping -t baidu.com` |
| `tracert` | 追踪网络路径 | `tracert baidu.com` |
| `netstat` | 查看网络连接 | `netstat -ano` |
| `netsh winsock reset` | 重置网络栈 | 需管理员权限，执行后重启电脑 |
| `ipconfig /release` | 释放当前 IP 地址 | `ipconfig /release` |
| `ipconfig /renew` | 重新获取 IP 地址 | `ipconfig /renew` |
| `nslookup` | 查询 DNS 记录 | `nslookup baidu.com` |
