# 权限与账户进阶：Windows 的安全机制

> 阅读前提：你已经了解了 UAC 弹窗的基本概念和应对方法（参见《用户账户控制与权限》）。本文讲解 Windows 权限的底层机制，供需要深入了解的读者参考。

---

## 一、访问令牌（Access Token）

### 什么是访问令牌

当你登录 Windows 时，系统会为你创建一个**访问令牌**。这个令牌就像一张"身份卡"，上面记录了：

- 你的用户身份（SID，安全标识符）
- 你属于哪些用户组（如 Administrators、Users）
- 你拥有哪些权限

之后你运行的每个程序，都会继承这张身份卡。Windows 根据这张卡来决定你能访问哪些文件、能执行哪些操作。

### 为什么管理员账户也会弹窗

即使你用的是管理员账户，你运行的程序默认也只会拿到"标准用户"级别的令牌。只有当程序确实需要管理员权限时，UAC 才会弹窗请求提升。

这种设计叫**权限分离**：日常操作用最少的权限，敏感操作才提升。这样即使恶意软件运行了，它一开始也只有标准用户权限，无法随意修改系统。

---

## 二、SYSTEM 账户

### 比管理员更高的权限

Windows 里有一个特殊的账户叫 **SYSTEM**（或 LocalSystem），它的权限比管理员还高：

- 管理员账户：可以安装软件、修改系统设置、管理其他用户
- SYSTEM 账户：可以访问所有文件（包括管理员也打不开的系统文件）、修改系统内核

SYSTEM 账户不是给人用的。它用于运行系统核心服务——比如 Windows 更新服务、安全中心服务。这些服务需要最高的权限才能正常工作。

### 为什么有些文件连管理员都删不掉

因为那些文件的所有者是 SYSTEM 账户。即使你以管理员身份运行，也没有权限修改 SYSTEM 拥有的文件。这是 Windows 的核心保护机制，防止系统文件被意外或恶意修改。

---

## 三、TrustedInstaller

### 另一个系统级账户

**TrustedInstaller** 是 Windows 引入的另一个特殊账户，专门用于保护系统文件。

很多系统文件的所有者不是 SYSTEM，而是 TrustedInstaller。这意味着：

- 连 SYSTEM 账户都不能随意修改这些文件
- 只有 TrustedInstaller 自己才能修改

这种设计的目的是防止恶意软件即使获取了 SYSTEM 权限，也无法篡改核心系统文件。

### 如果确实需要修改系统文件

有些高级操作（如替换系统主题、修改系统组件）需要修改 TrustedInstaller 拥有的文件。这时需要：

1. 获取文件的所有权（把所有者改成管理员）
2. 给自己添加修改权限
3. 修改文件

**警告**：这种操作风险很高，修改错误的系统文件可能导致系统无法启动。除非你确切知道自己在做什么，否则不建议尝试。

## 四、文件权限的详细机制

### NTFS 权限基础

Windows 的 NTFS 文件系统支持为每个文件或文件夹设置详细的访问权限。右键文件 → 属性 → 安全选项卡，可以看到权限列表。

常见的权限类型：

| 权限 | 说明 |
| -------- | ------ |
| 完全控制 | 可以读、写、修改、删除、改权限 |
| 修改 | 可以读、写、修改、删除 |
| 读取和执行 | 可以读文件内容、运行程序 |
| 读取 | 只能读 |
| 写入 | 可以写和修改 |

### 权限的继承

子文件夹和文件默认继承父文件夹的权限。这意味着：

- 如果你给一个文件夹设置了"拒绝读取"，里面的所有文件和子文件夹也会继承这个限制
- 你可以单独打破继承，给某个子文件夹设置不同的权限

### 有效权限的计算

当多个权限设置冲突时，Windows 按以下规则计算：

1. **拒绝优先**：如果某个组拒绝了读取，即使另一个组允许读取，最终也是拒绝
2. **显式优先**：直接设置在文件上的权限优先于继承的权限
3. **累加原则**：多个允许的权限会累加（一个组给读取，另一个组给写入，最终就是读取+写入）

## 五、修改文件权限

### 获取所有权

当你遇到"拒绝访问"且需要修改时：

1. 右键文件夹 → 属性 → 安全 → 高级
2. 点击"更改"所有者
3. 输入你的用户名，点击"检查名称" → 确定
4. 勾选"替换子容器和对象的所有者"
5. 确定

### 修改权限

获取所有权后：

1. 回到"安全"选项卡
2. 点击"编辑"
3. 添加你的用户，给予需要的权限
4. 确定

**注意**：修改系统文件或程序文件夹的权限可能导致系统不稳定。操作前建议创建系统还原点。

## 六、用户账户类型详解

### 标准用户 vs 管理员

| 能力 | 标准用户 | 管理员 |
| -------- | --------- | -------- |
| 运行程序 | 可以 | 可以 |
| 修改自己的文件 | 可以 | 可以 |
| 安装软件 | 需要管理员密码 | 需要 UAC 确认 |
| 修改系统设置 | 需要管理员密码 | 需要 UAC 确认 |
| 修改其他用户的文件 | 不可以 | 可以（需权限） |
| 修改系统文件 | 不可以 | 不可以（需要 SYSTEM） |

### 为什么建议日常使用标准用户

如果你创建一个标准用户账户用于日常上网、办公，管理员账户只在需要安装软件时使用：

- 恶意软件运行时也只会获得标准用户权限
- 无法修改系统设置、无法安装驱动、无法修改其他用户的文件
- 即使中毒，影响范围也大大缩小

设置方法：设置 → 账户 → 家庭和其他用户 → 添加账户 → 选择"我没有这个人的登录信息" → 添加一个没有微软账户的用户 → 类型选"标准用户"。

## 七、以管理员身份运行的底层原理

当你右键选择"以管理员身份运行"时，发生了什么：

1. Windows 创建一个新的进程
2. 这个新进程请求管理员级别的访问令牌
3. UAC 弹窗，询问你是否同意
4. 你点击"是"后，进程拿到管理员令牌，可以执行敏感操作

这就是为什么日常软件不需要管理员权限——它们不需要修改系统级别的设置。

## 八、实用操作补充

### 设置程序始终以管理员身份运行

有些程序每次运行都需要管理员权限，你可以设置它默认以管理员身份运行：

1. 右键程序（或快捷方式）→ 属性
2. 点击"兼容性"选项卡
3. 勾选"以管理员身份运行此程序"
4. 点击确定

建议**只在确实有需要的程序上这么设置**，避免对所有程序都这样做。

### 常见文件权限场景

**场景一：操作另一个用户文件夹里的文件**

每个用户的个人文件夹（`C:\Users\用户名`）默认只允许该用户访问。你用账户 A 登录时，无法访问账户 B 的文档、桌面等个人文件夹（会提示"拒绝访问"）。

这是隐私保护，不是故障。如果你确实需要访问另一个账户的文件，可以在文件资源管理器中右键点击文件夹 → 属性 → 安全 → 高级 → 更改权限。但这属于高级操作，操作不当可能导致其他用户无法正常使用。搜索"Windows 文件权限 修改"了解具体步骤，**操作前请谨慎**。

**场景二：文件是从另一台电脑或旧系统复制过来的**

文件上的权限记录的是原电脑上的账户标识，换到新电脑后自然对不上。Windows 会提示"拒绝访问"。

**解决方法：**

1. 右键文件 → 属性 → 安全 → 高级
2. 点击"更改"将所有者改为当前用户
3. 应用更改后，重新设置权限

搜索"获取文件所有权 Windows"了解更详细的操作步骤。

## 九、总结

| 概念 | 一句话理解 |
| -------- | ---------- |
| 访问令牌 | 你的"身份卡"，决定你能做什么 |
| UAC | 敏感操作前的确认弹窗 |
| SYSTEM | 系统级账户，比管理员权限还高 |
| TrustedInstaller | 保护核心系统文件的账户 |
| NTFS 权限 | 控制谁能对文件做什么 |
| 标准用户 | 日常使用的安全账户类型 |

对大多数用户来说，理解"UAC 弹窗时点哪个按钮"就够了。只有需要修改系统文件、排查权限问题、或者进行系统级开发时，才需要深入了解这些机制。
