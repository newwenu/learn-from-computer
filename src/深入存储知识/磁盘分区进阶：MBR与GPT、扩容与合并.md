## 一、MBR 与 GPT 的底层区别

### 为什么需要分区表

硬盘本身只是一块存储介质，操作系统需要知道"从哪里到哪里是第一个分区""第二个分区在哪里"，才能正确读写数据。分区表就是记录这些信息的"目录"。

### MBR 的局限

MBR（Master Boot Record，主引导记录）诞生于 1983 年，设计时没想到硬盘会超过 2TB：

- 使用 32 位来记录扇区地址，最大寻址空间约 2.2TB
- 分区表只有 64 字节，最多记录 4 个主分区
- 没有数据校验机制，分区表损坏后数据难以恢复

### GPT 的优势

GPT（GUID Partition Table，全局唯一标识分区表）是 Intel 在 1990 年代末提出的标准：

- 使用 64 位地址，理论支持 9.4 ZB（1 ZB = 10 亿 TB）
- 分区表在硬盘头部和尾部各存一份（冗余备份），一份损坏还能用另一份恢复
- 每个分区有唯一的 GUID 标识，避免分区表混乱

### BIOS/UEFI 与分区表的配对关系

| 固件类型 | 配合的分区表 | 说明 |
|---------|------------|------|
| BIOS（传统） | MBR | 2012 年前的老电脑 |
| UEFI（现代） | GPT | 2012 年后的新电脑 |

**为什么不能混用：** BIOS 只能读取 MBR 格式的分区表来找到操作系统；UEFI 则需要读取 GPT 分区表中的 EFI 系统分区来启动。如果配对错误，电脑会找不到系统，无法开机。

**如何判断自己的电脑是 BIOS 还是 UEFI：**

1. 按 Win+R，输入 `msinfo32` 回车
2. 找到"BIOS 模式"一行，显示"传统"就是 BIOS，显示"UEFI"就是 UEFI

## 二、合并非相邻空间

Windows 磁盘管理的"扩展卷"功能有一个限制：**只能向右合并紧邻的未分配空间**。

如果你的分区布局是：

```text
[C 盘][D 盘][未分配空间]
```

那么 D 盘可以向右扩展，但 C 盘无法跳过 D 盘去合并后面的未分配空间。

### 解决方案

**方案一：删除中间分区（数据需备份）**

1. 备份 D 盘所有数据
2. 删除 D 盘分区 → 变为"未分配"
3. 扩展 C 盘 → 合并全部未分配空间
4. 在剩余空间上重建 D 盘

**方案二：使用第三方分区工具（无需删除）**
工具如 DiskGenius、傲梅分区助手可以移动分区位置，把 D 盘整体向右挪，让未分配空间紧邻 C 盘右侧。但这类操作涉及大量数据搬移，**务必先备份**。

## 三、MBR 的扩展分区与逻辑分区

MBR 最多只能有 4 个主分区。如果需要更多分区，可以把其中一个主分区改为"扩展分区"，然后在扩展分区内部创建多个"逻辑分区"。

```text
[主分区1: C盘][主分区2: D盘][扩展分区]
                                    ├─ [逻辑分区1: E盘]
                                    ├─ [逻辑分区2: F盘]
                                    └─ [逻辑分区3: G盘]
```

这种设计是 MBR 时代的权宜之计，增加了分区表的复杂度。GPT 没有这种限制，所有分区都是"主分区"，最多 128 个。

## 四、分区操作的风险等级

| 操作 | 风险 | 说明 |
| ------ | ------ | ------ |
| 新建分区 | 低 | 在空白区域操作，不影响现有数据 |
| 格式化 | 高 | 清空该分区全部数据 |
| 删除分区 | 高 | 该分区数据全部丢失 |
| 收缩卷 | 中 | 一般不会丢数据，但极端情况下可能出错 |
| 扩展卷 | 低 | 只扩大分区边界，不移动数据 |
| 移动分区位置 | 高 | 涉及大量数据搬移，可能中断导致损坏 |
| 转换分区表格式（MBR↔GPT） | 高 | 通常需要清空整个硬盘 |

---

## 五、什么时候需要关心分区表

- **安装操作系统时**：需要确认分区表格式与电脑固件匹配
- **购买 4TB 以上硬盘时**：MBR 无法识别超过 2TB 的空间
- **从旧电脑拆硬盘装到新电脑时**：可能需要确认分区表兼容性
- **数据恢复时**：分区表损坏是常见故障之一，了解原理有助于判断恢复可能性

日常使用中，只要你的电脑是 2012 年后购买的，分区表选 GPT 就不会有问题。
