# Windows 存储进阶：C 盘里到底发生了什么

> 阅读前提：你已经掌握了文件管理的基本操作（参见《日常使用》），并了解过系统自带的清理工具。本文深入讲解 C 盘文件夹结构和系统文件原理，供想进一步了解存储机制的读者参考。

---

## 一、C 盘根目录下的关键文件夹

打开 C 盘（按 Win+E → 左侧点击"此电脑" → 双击 C 盘），你会看到几个文件夹。下面详细说明每个文件夹放的是什么。

> **看不到某些文件夹？** 部分系统文件夹默认是隐藏的。如果下面提到的文件夹你找不到，在文件资源管理器顶部点击"查看" → 勾选"隐藏的项目"即可显示。

### Program Files 和 Program Files (x86)

存放你安装的应用程序。

- **Program Files**：64 位程序安装在这里
- **Program Files (x86)**：32 位程序安装在这里

**为什么分两个？** Windows 有 32 位和 64 位两种程序。64 位系统能运行两种程序，但把它们分开存放更便于管理。32 位程序不知道 64 位系统的存在，单独放一个目录避免混淆。

大多数情况下你不需要直接进这两个文件夹。卸载软件应该通过"设置 → 应用 → 已安装的应用"来操作，不建议直接删除这里的文件夹——会留下注册表残留、快捷方式残留等问题。

### Windows

存放操作系统本身的核心文件。**不建议手动修改或删除这个文件夹里的内容**，系统运行全靠它。

里面包括：

- **System32**：系统核心文件、驱动程序、命令行工具。名字里有 32，但 64 位系统的核心文件也在这里（历史遗留名称）。
- **Temp**：系统和部分程序存放临时文件的地方
- **WinSxS**：系统组件存储，体积很大，存放着 Windows 的各种系统组件的不同版本，供系统恢复和兼容性调用

#### 关于 WinSxS

WinSxS（Windows Side by Side）是 Windows 的系统组件存储库。它的作用是：

- 存放系统文件的多版本副本，确保旧程序能调用到它们需要的版本
- 支持系统恢复——当更新导致问题时，可以回退到之前的版本
- 体积会随着使用时间增长，因为每次更新都会保留旧版本

**不建议手动删除 WinSxS 里的文件。** 虽然它很大，但 Windows 提供了安全的清理方式（磁盘清理 → Windows 更新清理）。手动删除可能导致系统无法启动或更新失败。

### Users（用户）

存放所有用户的个人数据。每个用户一个子文件夹，名字就是你的用户名。

### ProgramData

存放应用程序的共享数据和设置（不区分用户，对所有登录用户生效的配置）。默认是隐藏的。不需要手动操作。

## 二、你的用户文件夹（Users）

进入 `C:\Users\你的用户名`，你会看到系统为你创建的几个专用文件夹。

| 文件夹 | 路径名 | 用途 |
|--------|--------|------|
| **桌面** | Desktop | 桌面上所有文件和快捷方式 |
| **文档** | Documents | 文档、学习资料的默认保存位置 |
| **下载** | Downloads | 浏览器下载的文件默认存到这里 |
| **图片** | Pictures | 图片和照片 |
| **视频** | Videos | 视频文件 |
| **音乐** | Music | 音频文件 |

### AppData — 程序存放设置的地方

这是一个**隐藏文件夹**，存放每个程序的专属设置和缓存数据。它分为三个子文件夹：

- **Roaming**：会跟随微软账户同步到其他电脑的程序设置。换电脑登录同一账户，这些设置还在。
- **Local**：只属于这台电脑的程序设置和缓存。换电脑就不在了。
- **LocalLow**：给安全性要求最低的程序用的，普通用户几乎用不到。

在文件资源管理器地址栏输入 `%appdata%` 可以快速打开 Roaming 文件夹，输入 `%localappdata%` 打开 Local 文件夹。

日常使用建议：**不建议手动修改或删除**。AppData 里的文件体积可能很大（浏览器缓存、聊天软件的历史文件），但手动删除可能引发程序异常。如果你需要清理空间，最好是去对应程序的设置里操作（比如在浏览器设置里清除缓存）。

## 三、把用户文件夹改到其他盘

> 这是**预防 C 盘爆满最有效的办法之一**。与其等 C 盘满了再清理，不如让文件从一开始就不往 C 盘存。

Windows 允许你把"文档""下载""桌面"这几个特殊文件夹的默认位置改到其他盘。比如你电脑有 C 盘和 D 盘，就可以改到 D 盘；如果有 E 盘，也可以改到 E 盘。下面以改到 D 盘为例，其他盘的操作完全一样。

这样你日常存的文件、下载的安装包、放在桌面的东西，就不会占 C 盘空间了。

### 为什么要改

- **下载文件夹会无限增长**：浏览器下载的文件默认进"下载"，日积月累可能几十 GB
- **桌面其实是 C 盘的一部分**：很多人习惯把文件直接放桌面，其实桌面就是 `C:\Users\你的用户名\Desktop`，放多了 C 盘就满了
- **重装系统不丢文件**：如果系统崩溃需要重装，C 盘数据可能被清空。文件在其他盘就不受影响
- **比清理更治本**：清理是治标，改位置是治本

### 怎么改

以"文档"文件夹为例：

1. 按 `Win + E` 打开文件资源管理器，在左侧找到"文档"（或"下载""桌面"）
2. 右键点击 → 选择"属性"
3. 在弹出的窗口上方，找到一排标签，点"位置"
4. 你会看到当前路径（比如 `C:\Users\你的用户名\Documents`），点击"移动"
5. 在弹出的窗口中，选择你想移到的盘（比如 D 盘），然后点击"新建文件夹"，起个名字（比如 `文档`），选中它，点"选择文件夹"
6. 回到属性窗口，点"应用"
7. Windows 会弹出一个对话框，问"是否要将所有文件从原位置移动到新位置？"——点"是"
8. 等待移动完成，点"确定"

**对"下载"和"桌面"重复以上步骤即可。**

### 建议迁移的文件夹

| 文件夹 | 是否建议迁移 | 原因 |
|--------|-------------|------|
| **文档** | **强烈建议** | 日常文件积累最多 |
| **下载** | **强烈建议** | 浏览器下载默认位置，安装包堆积地 |
| **桌面** | **建议** | 很多人习惯把文件放桌面 |
| 图片 | 可选 | 如果照片很多可以迁 |
| 视频 | 可选 | 视频文件通常很大 |
| 音乐 | 可选 | 一般不大，可不改 |

### 注意事项

- **目标盘要有足够空间**：迁移前确认目标盘剩余空间大于你要迁移的文件夹大小
- **迁移过程中尽量避免中断**：不要关机、不要强制关闭窗口
- **迁移后检查两边**：打开原来的位置（C 盘）和新位置，确认文件都在新位置了。如果原位置还有残留的空文件夹，可以手动删除
- **快捷方式仍然有效**：迁移后，你之前创建的快捷方式、最近使用的文件列表，Windows 会自动更新路径，不需要重新设置
- **如果后悔了**：右键文件夹 → 属性 → 位置 → "还原默认值"，可以移回 C 盘

### 另一种方式：改浏览器和软件的默认保存路径

除了迁移系统文件夹，你还可以单独改常用软件的默认保存位置：

- **浏览器**：设置 → 下载 → 修改下载路径到其他盘
- **微信/QQ**：设置 → 文件管理 → 改到其他盘（聊天记录和接收的文件会存到新位置）
- **Office/WPS**：选项 → 保存 → 修改默认文件位置

这样即使不迁移系统文件夹，也能减少 C 盘压力。

### 系统级的默认保存位置

Windows 还提供了一个系统级的设置，可以统一修改"新内容"的默认保存盘：

设置 → 系统 → 存储 → 保存新内容的地方 `[查]`

这里可以分别设置新应用、新文档、新音乐、新照片、新视频的默认保存位置。改完后，支持这个功能的应用会自动把新内容存到新位置。

这与前面说的"迁移系统文件夹"不同：迁移是把你已有的文件夹搬到其他盘，而这个设置只影响"新内容"的默认位置。两者可以结合使用——先用系统设置改默认，再把已有文件夹迁移过去。

## 四、临时文件夹

Windows 和很多程序在工作时会产生临时文件。正常情况下程序用完就会自动删除，但偶尔会有残留。

有两个主要的临时文件夹：

- `C:\Windows\Temp` — 系统临时文件（需要有管理员权限才能清理）
- `C:\Users\你的用户名\AppData\Local\Temp` — 当前用户的临时文件（`%temp%` 是它的快捷路径）

这里的文件理论上可以全部选中删除（有些正在使用的文件会提示"无法删除"，跳过就行），但不需要特意去找——系统自带的清理工具会处理它们。

## 五、回收站

按 Delete 键删除的本地硬盘文件默认不会真正消失，而是被移到了回收站。回收站占用的是 C 盘空间，默认大小是 C 盘总容量的 5%-10%。当回收站里的文件总大小超过这个上限时，最旧的文件会被永久删除。

注意：U 盘、移动硬盘、网络位置等**可移动或远程存储上的文件**，按 Delete 会直接永久删除，不会进入回收站。

### 清空回收站

右键桌面回收站图标 → "清空回收站"。文件会彻底删除，无法从回收站恢复。

### 单独恢复或删除

双击回收站打开：

- 恢复某个文件：右键 → "还原"
- 彻底删除某个文件：右键 → "永久删除"

### 调整回收站大小

右键回收站 → "属性" → 在"自定义大小"中修改最大值。如果你 C 盘空间紧张，可以把比例调小一些。

---

## 六、自动清理：存储感知

Windows 自带的"存储感知"可以按设定周期自动清理临时文件和回收站内容。

### 如何开启

设置 → 系统 → 存储 → 存储感知 → 打开开关。 `[查]`

### 保守的设置建议

如果你不确定怎么设，可以这样选择：

- **运行存储感知**："每月"或"磁盘空间不足时"
- **删除临时文件**：开启
- **删除回收站文件**：选择"30天"或"60天"
- **删除下载文件夹文件**：**保持关闭**，避免误删还未处理的文件

### 它与磁盘清理的区别

- **存储感知**：自动运行，日常维护，"开了就不用管"
- **磁盘清理**：手动运行，深度清理，释放更多空间

建议两者结合：日常开着存储感知，C 盘空间紧张时再用磁盘清理。

## 七、系统文件详解

### pagefile.sys — 虚拟内存

当物理内存不够用时，Windows 会把一部分硬盘空间当作"虚拟内存"来使用。pagefile.sys 就是虚拟内存的文件。

- 它通常很大（几个 GB 到几十个 GB），由系统自动管理大小
- 不建议删除它——没有它，内存不足时程序会崩溃
- 如果 C 盘空间紧张，可以把它移到其他盘或限制大小：右键"此电脑" → 属性 → 高级系统设置 → 性能 → 设置 → 高级 → 虚拟内存 → 更改 `[查]`
- 空间充足时保持默认即可，由系统自动管理

### hiberfil.sys — 休眠文件

休眠功能会把内存中的所有数据写入硬盘，然后彻底断电。下次开机时从硬盘恢复内存数据，回到之前的工作状态。

- hiberfil.sys 的大小通常和内存容量相当（16GB 内存就有约 16GB 的休眠文件）
- 需要关闭时：用管理员权限打开命令提示符，输入 `powercfg /hibernate off`（参见《命令行入门》）。需要重新开启时输入 `powercfg /hibernate on`
- C 盘空间严重不足时，可以考虑关闭它释放空间。空间充足时保持默认即可，不必特意关闭

## 八、深度清理方法

### 磁盘清理的高级用法

在磁盘清理窗口中点击"清理系统文件"，可以看到更多选项：

- **Windows 更新清理**：删除旧版本的系统文件，通常能释放几个 GB
- **以前的 Windows 安装**：如果你最近升级过系统，旧版本会保留在这里以便回退
- **传递优化文件**：Windows 更新下载过程中暂存的缓存文件

**安全判断表格：**

| 项目 | 是否安全删除 | 说明 |
| ------ | --------- | ------ |
| 临时 Internet 文件 | **安全** | 浏览器缓存，删了不影响 |
| 回收站 | **安全** | 清空回收站 |
| 临时文件 | **安全** | 程序临时文件 |
| Windows 更新清理 | **安全** | 旧版本系统文件 |
| 传递优化文件 | **安全** | 更新缓存 |
| 缩略图 | **安全** | 自动重建 |
| 以前的 Windows 安装 | **谨慎** | 删除后无法回退旧版本 |
| 下载 | **谨慎** | 会清空整个下载文件夹 |

勾选完成后点击"确定"→"删除文件"。清单里的每一项都是系统判定可以安全删除的内容，勾选前确认你明白要删的是什么即可。

### 什么情况不需要清理

- C 盘还有几十 G 空余时，不必为了"优化"而频繁清理
- 电脑卡顿但 C 盘还有很多空间——卡顿通常不是垃圾文件导致的

### 手动清理下载文件夹

`C:\Users\你的用户名\Downloads` 是最容易堆积的地方。定期检查，删除不再需要的安装包和临时文件。

### 清理浏览器缓存

浏览器缓存可能占用几个 GB 空间。在浏览器设置中找到"清除浏览数据"，选择"缓存的图像和文件"进行清理。

### WinSxS 文件夹

`C:\Windows\WinSxS` 是系统组件存储目录，存放着系统文件的多个版本，用于更新回滚和系统修复。它会随时间增长，但**不建议手动删除其中的文件**——直接删除可能导致系统无法更新或修复。

**安全的清理方式：**

1. **磁盘清理**：勾选"Windows 更新清理"（见上文），可以删除已被取代的旧版本文件
2. **Dism 命令**（进阶）：用管理员权限运行命令提示符，输入 `Dism.exe /online /Cleanup-Image /StartComponentCleanup` 清理已被取代的组件。加上 `/ResetBase` 参数可以进一步清理，但清理后无法卸载之前的更新 `[查]`

如果系统很久没清理过，Dism 命令可能需要较长时间，甚至无法清理。这种情况比较复杂，暂时不在本文讲解怎么解决。

## 九、不建议做的事

| 操作 | 为什么不建议 |
| ------ | --------- |
| 手动删除 Windows 或 Program Files 里的文件 | 可能导致系统或程序无法正常运行 |
| 清理 AppData 里的文件夹 | 可能导致程序设置丢失或异常 |
| 删除 pagefile.sys 或 hiberfil.sys | 系统功能异常 |
| 使用第三方"清理大师"类软件 | 弹广告、捆绑软件、可能误删系统文件 |

---

## 十、总结

| 位置 | 用途 | 能手动清理吗 |
| ------ | ------ | --------- |
| Program Files | 已安装的程序 | 通过"设置→应用"卸载 |
| Users\你的用户名 | 个人文件 | 定期整理下载文件夹 |
| AppData | 程序设置和缓存 | 不建议手动清理，去程序设置里清理 |
| Windows\Temp | 系统临时文件 | 磁盘清理会自动处理 |
| Users\…\AppData\Local\Temp | 用户临时文件 | 可以手动删，但不必要 |
| 回收站 | 已删除文件的暂存 | 需要时清空 |
| pagefile.sys | 虚拟内存 | 可以移到其他盘，不建议删除 |
| hiberfil.sys | 休眠文件 | 不用休眠可以关闭 |
| WinSxS | 系统组件存储 | 用磁盘清理的"Windows更新清理" |

对大多数用户来说，**开启"存储感知"自动清理 + 把"文档""下载""桌面"改到 D 盘 + 偶尔运行一次"磁盘清理"** 就足够了。只有 C 盘空间严重不足时，才需要手动处理 pagefile.sys 和 hiberfil.sys。
