# 三大操作系统对比

> 阅读前提：你已经理解了 Windows 的基本概念（参见《认识PC》和《基本操作》）。本文供想了解其他主流操作系统差异的读者参考。

---

## 一、核心差异一览

| 概念 | Windows | macOS | Linux |
| ---- | ------------ | ------------ | ------------ |
| 用户文件夹路径 | `C:\Users\用户名`（中文版资源管理器显示为"用户"） | `/Users/用户名` | `/home/用户名` |
| 软件安装常见方式 | `.exe` 安装包、Microsoft Store | `.dmg` 安装包、App Store | 包管理器 |
| 常见文件系统 | NTFS、exFAT、FAT32 | APFS、Mac OS Extended | ext4 |
| 默认命令行 | PowerShell、cmd | zsh（bash 可选） | bash |

---

## 二、文件系统说明

- **APFS**（Apple File System）：macOS 的默认文件系统，针对固态硬盘优化，支持快照、空间共享等高级功能
- **Mac OS Extended**（HFS+）：macOS 的旧版文件系统，仍用于部分外置硬盘
- **ext4**（Fourth Extended Filesystem）：Linux 最常用的文件系统，支持日志、大文件、大容量分区

---

## 三、操作习惯差异

### 窗口控制按钮
- Windows：关闭 ×、最大化 □、最小化 — 在窗口右上角
- macOS：关闭 ×、最小化 —、全屏 □ 在窗口左上角（颜色为红黄绿）

### 键盘快捷键
- 复制：Windows 用 `Ctrl+C`，macOS 用 `Command+C`
- 粘贴：Windows 用 `Ctrl+V`，macOS 用 `Command+V`
- 撤销：Windows 用 `Ctrl+Z`，macOS 用 `Command+Z`

### 软件安装
- Windows：下载 `.exe` 或 `.msi` 安装包，双击运行安装向导
- macOS：下载 `.dmg` 文件，打开后将程序图标拖到"应用程序"文件夹
- Linux：通常使用包管理器命令安装（如 `apt install 软件名`）

---

## 四、迁移建议

它们的基本工作原理——操作系统管理硬件、运行程序、组织文件——是一致的，只是实现方式和操作逻辑不同。理解其中任何一个，迁移到另一个系统时只需要适应操作习惯的差异。
