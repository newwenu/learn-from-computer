# 术语表：全书概念速查

> 阅读前提：无。阅读主线或专题时遇到看不懂的词，可以随时来这里查。按分类排列，每条一句话说清是什么、为什么重要。

---

## 系统与权限

| 术语 | 一句话解释 | 相关章节 |
| ------ | ----------- | --------- |
| **注册表 (Registry)** | Windows 用来集中记录系统和软件设置的大型数据库。安装/卸载软件时都会读写它，所以不能直接删除软件文件夹（会残留注册表项） | 日常使用、故障排查 |
| **UAC (用户账户控制)** | Windows 的安全机制：即使你是管理员，程序默认也只有标准权限，需要修改系统设置时才弹窗让你确认 | UAC与权限 |
| **访问令牌 (Token)** | 你登录时 Windows 给你发的"身份证"，记录你是谁、有什么权限。你运行的每个程序都带着这个身份证 | UAC与权限 |
| **数字签名** | 程序的"电子身份证"，证明它确实来自某家公司且没有被篡改过。UAC 弹窗里的"发布者"一栏显示的就是签名信息 | UAC与权限 |
| **完整性级别** | Windows 给程序划分的信任等级。等级越低，程序能做的破坏性操作就越少，越安全 | Windows存储位置 |
| **SYSTEM 账户** | 比管理员权限还高的系统账户，Windows 内核和系统服务用它运行，普通用户接触不到 | UAC与权限 |
| **TrustedInstaller** | Windows 更新组件的保护账户，它锁着系统文件不让随便修改——所以有些文件连管理员也删不掉 | UAC与权限 |
| **驱动器号（盘符）** | Windows 给每个分区分配的字母标识（C:、D:、E:），相当于分区在系统中的"名字" | 日常使用、磁盘分区 |

## 存储与文件系统

| 术语 | 一句话解释 | 相关章节 |
| ------ | ----------- | --------- |
| **NTFS** | Windows 系统盘的默认格式。支持大文件（>4GB）、权限设置、加密，但 macOS 只能读不能直接写 | 理解OS、磁盘分区 |
| **FAT32** | 老格式，兼容性最好（电视、相机、游戏机都能认），但单个文件不能超过 4GB | 理解OS、磁盘分区 |
| **exFAT** | 专为 U 盘设计的格式，没有 4GB 限制，Windows 和 macOS 都能读写，适合传大文件 | 理解OS、磁盘分区 |
| **MBR (主引导记录)** | 老式分区表格式，最大支持 2TB 硬盘、最多 4 个主分区。2012 年前的旧电脑在用 | 磁盘分区 |
| **GPT (GUID 分区表)** | 现代分区表格式，支持 2TB 以上硬盘、分区数量几乎无限制。现在的新电脑都用的它 | 磁盘分区 |
| **UEFI** | BIOS 的现代替代品，启动更快、界面更友好、支持鼠标。2012 年后出厂的电脑基本都用的 UEFI | 磁盘分区 |
| **Legacy BIOS** | 传统电脑固件，文本界面、键盘操作，和 MBR 配合使用。新电脑上已基本淘汰 | 磁盘分区 |
| **BitLocker** | Windows 自带的磁盘加密功能，加密后的硬盘拆到别的电脑上也读不出来数据 | 磁盘分区、UAC与权限 |
| **休眠文件 (hiberfil.sys)** | 休眠时把内存数据存到硬盘的文件，体积和内存大小相当。如果不用休眠可以关掉它省空间 | 理解OS |
| **虚拟内存 (pagefile.sys)** | 硬盘划出一块区域假装是内存，当物理内存不够时系统把不活跃的数据挪到这里 | 理解OS |
| **RAID** | 把多块硬盘组合成一个逻辑单元，用来提高速度或做数据冗余备份 | 磁盘分区 |
| **VHD/VHDX** | Windows 的虚拟硬盘文件格式——一个文件里装着一整个"硬盘"，可以像物理硬盘一样分区和使用 | 磁盘分区 |

## 网络

| 术语 | 一句话解释 | 相关章节 |
| ------ | ----------- | --------- |
| **DHCP (动态主机配置协议)** | 设备连上路由器时自动获得 IP 地址的机制——所以插上网线就能用，不需要自己配 | 理解网络、日常使用 |
| **NAT (网络地址转换)** | 路由器把家里多台设备共享一个公网 IP 上网的技术——所有设备共用大门进出，路由器来分发 | 理解网络 |
| **运营商级 NAT (CGNAT)** | 运营商让多个宽带用户共享一个公网 IPv4 地址，你拿不到独立公网 IP。是目前家庭宽带的普遍情况 | 理解网络 |
| **公网 IP vs 内网 IP** | 公网 IP = 全球唯一地址（互联网上的"门牌号"）；内网 IP = 只在自家局域网有效（路由器分配的"工位号"） | 理解网络 |
| **端口** | IP 地址定位到设备，端口号定位到设备上的具体服务（相当于门牌号之后的房间号） | 理解网络 |
| **SSID** | Wi-Fi 网络的名称，就是你在手机或电脑上看到的 Wi-Fi 列表里的那个名字 | 理解网络 |
| **延迟 (Ping)** | 数据包从你的设备到目标再返回的时间，单位毫秒。决定打游戏顺不顺、网页打开快不快 | 理解网络 |
| **带宽** | 单位时间内能传输的数据量，像水管粗细。决定下载速度快不快 | 理解网络 |
| **Mbps vs MB/s** | 1 MB/s = 8 Mbps。运营商说的"100 兆宽带"实际下载速度约 12.5 MB/s，差 8 倍 | 理解网络 |

## 硬件

| 术语 | 一句话解释 | 相关章节 |
| ------ | ----------- | --------- |
| **超线程 (Hyper-Threading)** | Intel 的技术——让 CPU 的一个物理核心同时处理两路任务，在任务管理器里看到的"逻辑处理器"数通常是核心数的两倍 | 认识PC |
| **逻辑处理器** | 操作系统看到的"虚拟核心"。有超线程的 CPU，逻辑处理器数 = 物理核心数 × 2 | 认识PC、查看配置 |
| **寻道时间** | 机械硬盘的磁头移动到数据所在位置的时间，几毫秒到十几毫秒。固态硬盘没有这个部件所以快得多 | 认识PC |
| **DirectX** | Windows 的图形接口标准，游戏和图形软件通过它和显卡沟通。dxdiag 可以查看当前版本 | 查看配置 |
| **GPU (图形处理器)** | 显卡上的核心芯片，专门负责图像计算。分集成在 CPU 里的（集成显卡）和独立板卡的（独立显卡） | 认识PC |
| **缓存 (Cache)** | 硬盘/CPU/浏览器临时存放最近用过的数据，下次再用时不用重新加载，速度更快 | 认识PC、浏览器 |
| **DDR4 / DDR5** | 内存的规格代际。DDR5 更新、更快，但需要主板和 CPU 都支持才能用 | 认识PC |
| **时钟频率 (GHz)** | CPU 每秒钟执行的指令节奏。3.0GHz 表示每秒约 30 亿个周期，频率越高通常运算越快 | 认识PC |

## 软件与操作系统

| 术语 | 一句话解释 | 相关章节 |
| ------ | ----------- | --------- |
| **Shell（命令行界面）** | 和操作系统"打字聊天"的界面。Windows 自带的叫 cmd 或 PowerShell，macOS/Linux 叫 bash/zsh | 理解OS、命令行 |
| **PowerShell** | 比 cmd 更强大的 Windows 命令行工具，蓝色背景，功能类似 Linux 的 shell | 命令行入门 |
| **进程** | 你双击运行程序后，操作系统在内存里给它分配资源，此时它就成了一个"进程"——正在运行的程序实例 | 理解OS |
| **后台服务** | 不需要窗口、系统启动时就自动运行的特殊进程，比如 Windows Update、打印服务 | 理解OS |
| **驱动程序** | 操作系统和硬件之间的"翻译官"，让系统知道怎么用你的显卡、网卡、声卡等硬件 | 理解OS |
| **Cookie** | 网站留在你浏览器里的小文件，用来记住你的登录状态、偏好设置等 | 浏览器入门 |
| **缓存 (浏览器)** | 浏览器临时存下来的网页文件（图片、样式等），再次访问时直接读缓存不用重新下载 | 浏览器入门 |
| **扩展名** | 文件名最后一个小数点后面的部分（如 .docx、.pdf、.exe），标识文件类型 | 日常使用 |
