## 一、管道：让命令协同工作

你已经会用 `systeminfo` 和 `findstr`。如果把它们连起来：

```cmd
systeminfo | findstr "操作系统名称"
```

`|` 叫**管道符**，作用是把左边命令的输出传给右边处理。就像流水线上的传送带——`systeminfo` 产出大量信息，`findstr` 从中挑出你要的那一行。

```cmd
tasklist | findstr "chrome"          ← 从所有进程里找出 Chrome
ipconfig | findstr "IPv4"            ← 只看 IPv4 地址
```

**核心思路**：先用一个命令产生大量信息，再用 `findstr` 过滤。不用盯着满屏文字找。

## 二、重定向：把输出保存到文件

命令的结果默认显示在屏幕上。想存成文件，用这两个符号：

| 符号 | 作用 |
|------|------|
| `>` | 覆盖写入文件 |
| `>>` | 追加到文件末尾 |

```cmd
ipconfig /all > 网络配置备份.txt
```

执行后屏幕上什么都不显示，但当前目录会多出一个文件，里面完整记录了网络配置。以后网络出问题，可以对比这个备份。

## 三、通配符：批量匹配文件

| 符号 | 含义 | 示例 |
|------|------|------|
| `*` | 匹配任意字符 | `*.pdf` 匹配所有 PDF |
| `?` | 匹配单个字符 | `file?.txt` 匹配 file1.txt |

```cmd
copy C:\Users\你的用户名\Downloads\*.pdf D:\备份\
```

一句话把下载文件夹里所有 PDF 复制到备份盘。图形界面里要一个个选，命令行一句话搞定。

## 四、更多实用命令

```cmd
type 日志.txt                   ← 查看文本文件内容
tasklist | findstr "chrome"     ← 查看进程
tracert baidu.com               ← 追踪网络路径
nslookup baidu.com              ← 查询域名 IP
```

```cmd
copy 报告.txt D:\备份\         ← 复制
move 旧位置\文件.txt 新位置\   ← 移动
ren 旧名字.txt 新名字.txt       ← 重命名
```

```cmd
taskkill /im notepad.exe /f     ← 强制结束进程（确保进程名正确，误杀系统进程可能导致不稳定）
```

```cmd
sfc /scannow                    ← 扫描并修复系统文件（需管理员权限）
chkdsk C: /f                    ← 检查磁盘错误（执行前保存文档，修复可能需要重启）
```

```cmd
netsh winsock reset             ← 重置网络栈（执行后需重启）
```

## 五、批处理脚本：把命令写成文件

一组命令需要反复执行？写进 `.bat` 文件，以后双击就能跑。

```cmd
@echo off
echo 开始备份 PDF 文件...
copy C:\Users\你的用户名\Downloads\*.pdf D:\备份\
echo 备份完成！
pause
```

把上面内容保存为文本文件，后缀改成 `.bat`，双击运行。

- `@echo off`：不显示命令本身
- `echo`：输出提示文字
- `pause`：运行完暂停，显示"请按任意键继续..."

## 六、环境变量：为什么输入 ping 就能运行

输入 `ping`，Windows 怎么知道去哪找这个程序？因为它会在 `PATH` 里列出的文件夹中搜索：

```cmd
echo %PATH%
where ping
```

`where` 可以查出某个命令具体在哪个文件夹。`ping.exe`、`ipconfig.exe` 这些系统命令都在 `C:\Windows\System32` 里，所以你在任何目录都能直接运行。

## 七、总结

| 概念/命令 | 作用 | 典型场景 |
|----------|------|---------|
| `\|` 管道 | 命令输出传给下一个命令 | `systeminfo \| findstr "内存"` |
| `>` / `>>` | 输出保存到文件 | `ipconfig > 备份.txt` |
| `*` / `?` | 批量匹配文件 | `copy *.pdf D:\备份\` |
| `copy` / `move` / `ren` | 复制、移动、重命名 | 批量整理文件 |
| `tasklist` / `taskkill` | 查看和结束进程 | 程序卡死时强制关闭 |
| `sfc /scannow` | 修复系统文件 | 系统功能异常时 |
| `chkdsk` | 检查磁盘错误 | 文件频繁出错时 |
| `tracert` | 追踪网络路径 | 定位网络故障节点 |
| `nslookup` | 查询域名 IP | 判断 DNS 是否正常 |
| `netsh` | 网络配置与重置 | 重置网络栈 |
| `.bat` 脚本 | 批量执行命令 | 重复性操作的自动化 |
| `%PATH%` | 命令查找路径 | 理解命令为什么能直接运行 |

## 八、自行探索

1. **`命令 /?`** — 任何命令后面加 `/?` 都能查看帮助。比如 `taskkill /?`、`netsh /?`。

2. **PowerShell** — Windows 自带的另一个命令行工具（蓝色背景），功能更强大。日常简单任务 cmd 够用，需要自动化处理大量数据时再考虑它。
