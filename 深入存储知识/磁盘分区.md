# 磁盘分区：把一块硬盘分成多个独立空间

## 一、分区是什么

一块物理硬盘可以划分成多个逻辑区域，每个区域称为一个"分区"。每个分区在操作系统中被识别为一个独立的存储空间，互不干扰。

一个常见的比喻：硬盘就像一栋大楼，分区就是楼里独立隔开的房间。大楼本身没变，但里面的空间被划分成了各自独立的功能区。你在一个房间里做的事情不会影响另一个房间。

分区的信息（从哪里开始、到哪里结束）记录在硬盘最开头的一块区域里，称为**分区表**。

---

## 二、分区的实际用途

**系统与数据分离。** 操作系统安装在 C 盘，个人文件放在 D 盘。如果需要重装系统，可以直接格式化 C 盘重装，D 盘的文件不受影响。这是分区最常见的用途。

**多系统共存。** 在一台电脑上安装 Windows 和 Linux 双系统时，各自占用一个独立分区，互不覆盖。开机时选择进入哪个系统。

**多硬盘管理。** 电脑有多块硬盘时，每块硬盘可以独立分区或不分区直接使用。Windows 会给每个分区分配一个盘符（C:、D:、E:……），方便识别。

**性能与管理的考量。** 分区本身不会让硬盘变快，但可以避免一个分区写满导致整个硬盘空间紧张。文件按功能区分区存放也更便于管理。

---

## 三、分区表格式：MBR 与 GPT

分区表是硬盘前端的特殊数据区，记录分区的布局信息。有两种主流格式：

### MBR（主引导记录）

- 传统格式，1980 年代沿用至今
- 单个分区最大支持 **2TB**。超过 2TB 的硬盘用 MBR 格式无法利用全部空间。（进阶：4K 扇区硬盘下 MBR 的分区上限不同，但 Windows 环境下以 2TB 为准——有兴趣的可通过搜索引擎或 AI 了解详情）
- 最多支持 **4 个主分区**（或 3 个主分区 + 1 个扩展分区，扩展分区内可再划分多个逻辑分区）
- 与 Legacy BIOS 引导方式配合使用

### GPT（GUID 分区表）

- 现代格式，逐渐取代 MBR
- 单个分区最大支持 **9.4ZB**（1ZB = 10 亿 TB），当前所有硬盘都不会触及这个上限
- 分区数量理论上没有限制（Windows 限制为 128 个）
- 分区表在硬盘上有冗余备份，容错性更好
- 与 UEFI 引导方式配合使用

**如何判断：** 打开"磁盘管理"（右键开始菜单 → 磁盘管理），右键磁盘左侧区域 → "属性" → "卷"选项卡，可以查看分区表类型。2012 年后出厂的品牌电脑基本都使用 GPT。

### 需要注意

MBR 和 GPT 是分区表的格式，和文件系统（NTFS、FAT32、exFAT）是两个不同层面的概念。分区表描述硬盘怎么分区，文件系统描述分区里的数据怎么组织。

---

## 四、文件系统：分区里怎么组织数据

创建分区后还需要**格式化**——在分区上建立文件系统，才能往里存文件。文件系统决定了单个文件不能超过多大、支不支持权限设置、其他操作系统能否识别等。

### NTFS

Windows 系统盘和大多数数据盘的默认格式。支持单个文件超过 4GB、支持权限设置和加密、支持日志（意外断电后更容易恢复数据）。macOS 只能读取 NTFS（不能直接写入），Linux 需要额外软件才能写入。

### exFAT

专为 U 盘和存储卡设计，突破了 FAT32 的 4GB 文件大小限制，兼容性较好（Windows 和 macOS 都支持读写）。适合需要在不同操作系统之间传输大文件的 U 盘。

### FAT32

较旧的格式，兼容性最好（游戏机、电视、相机、车机等设备基本都能识别），但单个文件不能超过 4GB。多数 U 盘出厂时是这种格式。

**日常选择：**

- 内置硬盘 → NTFS
- U 盘 / 移动硬盘，只在 Windows 用 → NTFS
- U 盘 / 移动硬盘，需要 Windows 和 Mac 之间传文件 → exFAT
- 给老旧设备（相机、电视）用 → FAT32

（进阶：Linux 常用 ext4、Btrfs、XFS 等文件系统，和 Windows 不互通。如果你想研究不同文件系统的性能对比和适用场景，可以后续通过搜索引擎或 AI 进一步了解。）

## 五、Windows 下的典型分区布局

厂商预装系统的电脑通常按以下方式分区：

| 分区 | 用途 | 说明 |
| ------ | ------ | ------ |
| EFI 系统分区 | 存放引导程序 | GPT 硬盘必需，约 100MB，不分配盘符 |
| C 盘 | 操作系统和软件 | 系统分区，占用主要空间 |
| 恢复分区 | 系统恢复环境 | 用于重置或修复系统，不分配盘符 |
| D 盘（如有） | 数据分区 | 存放个人文件 |

这些隐藏分区（没有盘符的分区）是系统正常运行的一部分，在"磁盘管理"中可以看到它们，**不需要去动它们**。

## 六、操作分区的基本思路

分区操作涉及的风险因操作类型而异，先区分两种情况：

**常规操作（Windows 磁盘管理可完成）：** 新建分区、删除分区、格式化、收缩/扩展卷。这些操作只在分区表末尾附加或移除记录，不涉及搬移已有数据（收缩卷例外——它会把尾部数据向安全区移动，但仍在同一分区内）。只要操作不中断，数据风险很低。操作前建议简单确认一下：当前分区有没有还需要的数据、笔记本是否需要接上电源，心里有数就行。

**进阶操作（通常需要第三方工具）：** 移动分区位置、从两个分区间挤出一块新空间、在不删除分区的情况下合并非相邻空间。这些操作需要重写分区表的大段内容，甚至涉及跨分区搬移数据，复杂度和风险都高于常规操作。进行这类操作前，建议：

- 提前备份重要数据（复制到另一块盘或 U 盘即可）
- 确保电源稳定（笔记本插上电源）
- 先在搜索引擎或 AI 中搜索具体操作步骤，理解每一步做什么再动手，不要边试边学

**一点补充：** 调整盘符（比如把蓝光光驱的 D 盘改成 E 盘）不涉及修改分区表，只是改个标签，没有数据风险。在"磁盘管理"中右键分区 → "更改驱动器号和路径"即可。

## 七、实际操作：Windows 磁盘管理

Windows 自带的"磁盘管理"工具可以完成绝大多数分区任务。以下是几种常用操作的步骤。

### 打开磁盘管理

四种方式均可：

- 右键开始菜单（Win+X）→ 磁盘管理
- 按 Win+R 输入 `diskmgmt.msc` 后回车
- 设置 → 存储 → 磁盘管理
- 右键"此电脑" → 管理 → 磁盘管理

### 查看分区信息

窗口上半部分显示现有各分区的盘符、文件系统、容量、状态。下半部分以图形方式显示每块硬盘上的分区布局，能直观看到哪些分区是相邻的、哪些区域是空闲的。

### 创建一个新分区

场景：硬盘上有未分配空间，想建一个新分区来存放文件。

1. 右键该未分配区域 → "新建简单卷"
2. 按向导操作：指定大小 → 分配盘符 → 选择文件系统（NTFS）→ 完成

整个过程不需要重启。

### 格式化分区

**格式化会清空该分区上的所有文件。** 操作前确认该分区里的数据已不需要，或已备份到其他盘。

确认无误后，右键该分区 → "格式化"，按向导完成即可。

### 收缩卷（缩小分区）

场景：C 盘太大，想分出一些空间给 D 盘。

1. 右键 C 盘 → "收缩卷"
2. 输入要缩小的容量（单位 MB），点击"收缩"
3. 操作完成后，C 盘右侧会出现一块"未分配"空间

### 扩展卷（扩大分区）

场景：C 盘太小，想把空闲空间并入 C 盘。

1. C 盘右侧必须有紧邻的"未分配"空间（不是"空闲"或别的分区的空间）
2. 右键 C 盘 → "扩展卷"，按向导操作

如果未分配空间不在 C 盘右侧，而是隔着一个分区或者在某分区的右侧，Windows 磁盘管理无法跨分区扩展。这种情况要么用第三方工具，要么参考前文"给 C 盘扩容"一节删除中间分区（先备份数据）后再扩展。

### 删除分区

**删除分区会使该分区上的所有数据丢失。** 删除前反复确认没有你需要的数据，如有需要先复制到其他盘或 U 盘。

确认无误后，右键该分区 → "删除卷"。删除后该分区变为"未分配"空间。

### 更改驱动器号和路径

右键分区 → "更改驱动器号和路径"→"更改"。可以在弹出的窗口中选择一个未被占用的字母作为新的盘符，也可以选择"装入以下空白 NTFS 文件夹中"（将一个分区挂载到另一个分区的文件夹路径下——进阶操作，有兴趣可进一步了解）。

### 初始化新硬盘

首次接入一块全新的硬盘（或刚清空分区表的硬盘）时，Windows 会提示"初始化磁盘"。你需要选择分区表格式。判断依据如下：

- 如果这块硬盘要**安装操作系统**（作为系统盘），分区表格式必须和电脑的引导方式匹配。现在绝大多数电脑都是 **UEFI + GPT**（2012 年后出厂的基本都是），选 GPT 即可。只有极少数老旧设备（2010 年以前的电脑安装 XP 等）才需要用 Legacy BIOS + MBR。
- 如果这块硬盘**只存放数据**（不装系统），优先选 **GPT**。GPT 没有 2TB 容量限制，兼容性也足够好（Windows Vista 之后的系统都能正常读写 GPT 数据盘）。只有需要把硬盘接到不识别 GPT 的设备（如老旧电视、车载播放器、Windows XP 等）时才选 MBR。

如果选错了可以重新初始化（重新选格式），但会清空硬盘上的所有数据。

## 八、常见分区场景

### 新硬盘怎么分区

一块全新硬盘在 Windows 中初始化为 GPT 后，可以先创建分区作为系统盘（用于安装操作系统），剩余空间可以暂时不分区，系统安装完成后在"磁盘管理"中继续划分。

如果硬盘作为数据盘使用，可以直接创建一个或多个分区，格式化后分配盘符即可。

### 给 C 盘扩容

C 盘空间不够时，如果 D 盘有富余空间且数据可以转移，一种简单做法是：备份 D 盘全部数据，删除 D 盘分区，使空闲区域紧邻 C 盘右侧，然后在"磁盘管理"中右键 C 盘选择"扩展卷"。扩展完成后，重新创建 D 盘分区并将数据拷回。

如果不想删除 D 盘（保留 D 盘数据同时缩小它来给 C 盘腾空间），Windows 自带的磁盘管理无法直接完成，因为"收缩卷"空出的空间出现在被收缩分区的最右侧，而不是 C 盘旁边。这种情况需要第三方工具（如 DiskGenius、傲梅分区助手）来处理。

### 合并多余分区

如果有两个相邻的分区（比如 D 盘和 E 盘）想合并成一个，先把其中一个分区的数据转移到别处，删除该分区使其变为"未分配"空间，然后右键相邻分区选择"扩展卷"。

### BitLocker 加密需要注意

如果开启了 BitLocker 设备加密（Windows 10/11 的某些版本默认开启），调整分区前需要先挂起或关闭 BitLocker，否则操作后分区可能无法访问。可以在"控制面板 → BitLocker 驱动器加密"中查看和管理。操作完成后再重新开启。

### 进阶场景（有兴趣可后续了解）

- **动态磁盘与跨区卷**：可以创建跨多个硬盘的卷（把一个分区跨到多块物理硬盘上），不常用，已逐步被存储空间功能取代。
- **VHD/VHDX**：Windows 支持创建虚拟硬盘文件，就像操作一个物理硬盘一样分区和使用。适合测试系统或隔离工作环境。
- **RAID**：多块硬盘组合成一个逻辑单元，提高性能或容错。通常由主板或独立的 RAID 卡管理。
- **数据恢复**：误删分区后，只要没有覆盖写入新数据，用专业工具可能找回分区和文件。但这属于数据恢复领域，需要专门的知识。

以上场景通过搜索引擎或 AI 可以找到详细操作说明，在实际动手前建议先查阅确认。

## 九、小结

| 概念 | 要点 |
| ------ | ------ |
| 分区 | 一块硬盘划分成的独立逻辑区域 |
| 分区表 | 记录分区布局的数据，MBR（旧）或 GPT（新） |
| 文件系统 | 管理分区内数据的方式，如 NTFS、exFAT、FAT32 |
| 盘符 | Windows 给分区分配的字母标识（C: D:） |
| 系统分区 | 存放操作系统的分区，通常是 C 盘 |
| 数据分区 | 存放个人文件的分区，通常是 D 盘等 |

日常使用中，绝大多数人不需要操作分区。当你确实需要时：

1. 先备份重要数据
2. 优先使用 Windows 磁盘管理
3. 操作中保证电源稳定、不中途中断
4. 遇到超出基础操作的需求，先在搜索引擎或 AI 中确认步骤再动手
